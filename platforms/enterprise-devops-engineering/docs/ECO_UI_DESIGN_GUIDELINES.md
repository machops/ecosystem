# ECO 平台 UI/UX 設計指南

## 1. 概述

本文件旨在為 ECO 平台生態系統建立一套統一、專業且高效的 UI/UX 設計標準。透過融合業界領先的設計實踐（如 Exness 的商業清晰度與 SonarCloud 的技術專業感），我們旨在打造一個既美觀又實用的企業級介面。

## 2. 設計原則

- **一致性 (Consistency)**：所有平台與模組應遵循統一的視覺風格與互動模式。
- **清晰度 (Clarity)**：介面應直觀易懂，關鍵資訊與操作應一目了然。
- **專業感 (Professionalism)**：設計應體現企業級軟體的嚴謹性與可靠性。
- **效率 (Efficiency)**：工作流程應被優化，減少用戶點擊與認知負擔。

## 3. 核心視覺系統 (Core Visual System)

### 3.1 色彩系統

色彩系統旨在建立清晰的視覺層次，引導用戶注意力並傳達狀態資訊。

| 用途 | 顏色 | HEX 色碼 | 備註 |
| :--- | :--- | :--- | :--- |
| **主要操作 (Primary)** | 金黃色 | `#FFD700` | 用於關鍵號召性用語 (CTA)，如「創建」、「部署」。 |
| **輔助/資訊 (Secondary)** | 科技藍 | `#1E88E5` | 用於次要按鈕、連結與資訊性元素。 |
| **背景 (Background)** | 純白色 | `#FFFFFF` | 提供乾淨、開闊的版面基礎。 |
| **深色背景 (Dark BG)** | 深藍黑 | `#0A1628` | 用於頁尾、特定面板或高對比度區域。 |
| **主要文字 (Primary Text)** | 深藍灰 | `#1a1a2e` | 用於所有主要內文與標題。 |
| **次要文字 (Secondary Text)** | 中灰色 | `#666666` | 用於輔助說明、提示文字。 |
| **邊框/分隔線 (Borders)** | 淺灰色 | `#e5e7eb` | 用於卡片、輸入框邊框與分隔線。 |
| **危險/錯誤 (Danger/Error)** | 紅色 | `#FF4D4F` | 用於錯誤提示、刪除確認等危險操作。 |
| **成功 (Success)** | 綠色 | `#52C41A` | 用於成功狀態提示。 |

### 3.2 字體規範

- **主要字體**: 採用系統預設的無襯線字體（如 San Francisco, Segoe UI, Roboto），確保在各平台上的可讀性與一致性。
- **層次**: 透過字重 (Font Weight) 和字號 (Font Size) 的組合來區分資訊層級，避免過度依賴顏色。

## 4. 組件設計 (Component Design)

### 4.1 按鈕 (Buttons)

| 類型 | 樣式 | 互動效果 (Hover) |
| :--- | :--- | :--- |
| **主要按鈕 (Primary)** | 金黃色背景，深色文字 | 輕微陰影，顏色加深 |
| **次要按鈕 (Secondary)** | 白色背景，藍色邊框與文字 | 背景變為淺藍色 |
| **文字按鈕 (Text)** | 無背景/邊框，藍色文字 | 文字下方出現底線 |
| **危險按鈕 (Danger)** | 紅色背景，白色文字 | 顏色加深 |

- **圓角**: 所有按鈕統一使用 `8px` 的圓角半徑。

### 4.2 卡片 (Cards)

卡片是組織資訊的基本容器。

- **背景**: 純白色 (`#FFFFFF`)
- **邊框**: `1px` 實線，淺灰色 (`#e5e7eb`)
- **圓角**: `12px`
- **陰影**: 輕微的 box-shadow，以增加立體感。
- **懸停效果**: 陰影加深，卡片輕微上浮。

### 4.3 導航欄 (Navigation)

- **佈局**: Logo 靠左，導航連結居中，主要 CTA 按鈕（如「新增專案」）與用戶頭像靠右。
- **背景**: 白色背景，底部有 `1px` 的淺灰色分隔線。

## 5. 佈局與結構

- **網格系統**: 採用 12 欄位的響應式網格系統，確保在不同螢幕尺寸下的佈局一致性。
- **資訊展示**: 對於複雜的技術資訊（如代碼分析、監控數據），應優先採用專業的圖表與表格展示，並提供清晰的標籤與圖例。
- **產品展示**: 在需要展示具體功能時，可使用產品介面截圖或 Mockup，以增強場景感。
